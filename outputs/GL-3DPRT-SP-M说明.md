# GL-3DPRT-SP-M 运动范围判定说明

版本：V1.2 · 2026-07-03 · ©Ming Xia

## 这个工具是做什么的

这个工具用来判断一个长方体能不能放进 GL-3DPRT-SP-M 的运动范围里。

简单说：

- 绿色透明区域表示设备能到达的空间。
- 黄色长方体表示你输入的尺寸。
- 如果长方体完全在绿色范围内，就显示“长方体在范围内。”
- 如果有一部分超出去，就会提示不可达。

## 怎么看这个 3D 图

- 透明绿色体：设备可达范围
- 黄色长方体：当前输入的长方体
- 红色长方体：当前尺寸或位置不可达
- 淡黄色设备模型：设备外形参考，只是帮助看方向和位置，不参与判定
- X 方向：左右方向
- Y 方向：前后方向
- Z 方向：高度方向
- Z=0：地面

鼠标操作：

- 拖动鼠标：旋转视角
- 滚轮：放大或缩小
- 右上角“视图”：快速切换默认、前、上、右视图

左侧顶部有“深色 / 亮色”主题切换。亮色主题使用浅色面板、浅灰网格和红色强调按钮，选择会保存在当前浏览器中。

当前版本使用按需加载的设备模型文件。网页打开时不会下载、加载或解码模型，只显示运动范围、长方体和 UI。

需要查看设备外形时，点击左侧的“显示设备模型”。模型加载完成后，按钮会变成“隐藏设备模型”。这个设备模型只是参考外形，不参与可达范围判定，也不会影响最大贴合结果。

如果通过 GitHub Pages 或本地 HTTP 服务访问网页，按钮会加载 `outputs/assets/GL-3DPRT-SP-M.glb`。如果直接双击 HTML 文件、本地以 `file://` 方式打开，浏览器不能直接读取 GLB 文件，网页会改为加载 `outputs/assets/GL-3DPRT-SP-M-model.js` 备用模型包。

设备模型显示姿态：`rotation.x = -Math.PI / 2`，`rotation.y = -Math.PI / 2`，`rotation.z = Math.PI`。模型位置按包围盒居中，Y 方向基准为 `2500`，底部对齐到 `Z=0`。

## 导入 STL 参考模型

左侧有“STL参考模型”区块，可以导入本地 `.stl` 文件。

导入后可以调整：

- X / Y / Z 位置
- X / Y / Z 旋转
- 缩放
- 透明度

STL 文件只在当前浏览器本地读取，不会上传。导入的 STL 只作为临时观察参考，不参与运动范围判定，也不会影响最大贴合结果。

“重置参数”会把 STL 的位置、旋转、缩放和透明度恢复默认值。“清除 STL”会从场景中移除当前导入的 STL 模型。

导入 STL 后，可以点击“显示交线”显示 STL 与运动范围边界的近似交线，再点击“隐藏交线”关闭。交线根据 STL 三角面边上“范围内/范围外”变化的位置生成，是视觉辅助，不参与判定。

## 隐藏或显示长方体

点击左侧“隐藏长方体”可以临时隐藏中间的内接长方体和尺寸标注，按钮会变成“显示长方体”。这个操作只影响显示，不会改变输入尺寸、位置、范围判定或最大贴合结果。

## 左侧参数怎么填

### X 向位置

X 方向固定在 `0`，也就是长方体左右居中放置。

这个值不能修改。

### Y 向内侧边位置

这个值表示长方体靠近设备内侧的那一条边在 Y 方向的位置。

- 如果 Y 是正数，长方体从这个位置向 Y+ 方向展开。
- 如果 Y 是负数，长方体从这个位置向 Y- 方向展开。

### 底面高度 Z

底面高度固定为 `0`，也就是长方体放在地面上。

长方体只向 Z+ 方向，也就是向上展开。

### L / W / H

- L：长方体在 X 方向的长度
- W：长方体在 Y 方向的宽度
- H：长方体在 Z 方向的高度

当前最大输入值：

- L 最大值：`20000 mm`
- W 最大值：`12000 mm`
- H 最大值：`10000 mm`

默认尺寸为：

```text
L x W x H = 2000 x 2000 x 2000 mm
```

## 最大贴合按钮

左侧有三个最大贴合按钮：

- 固定L最大贴合
- 固定W最大贴合
- 固定H最大贴合

它们的意思是：

- 固定L最大贴合：保持当前 L 不变，先找最大的 W，再找最大的 H。
- 固定W最大贴合：保持当前 W 不变，先找最大的 L，再找最大的 H。
- 固定H最大贴合：保持当前 H 不变，搜索更合适的 L/W 底面积组合。

最大贴合只是在当前设备运动范围内寻找一个合适长方体，不代表唯一解。

## 当前设备范围参数

### XY 平面

水平可达范围为：

- 外圆：圆心 `(0, 0)`，半径 `10187 mm`
- 内圆：圆心 `(0, -1118)`，半径 `3115.5 mm`

内圆区域按不可达区域处理，相当于一个沿 Z 方向贯穿的不可达圆柱。

### R-Z 垂直截面

垂直截面使用以下实测点生成，并做平滑处理：

```text
(R,Z)
(1600,0)
(10187,0)
(10129,1467)
(9904,3075)
(9416,4651)
(8700,6009)
(7598,7492)
(6809,8442)
(6192,9175)
(5861,9472)
(5549,9602)
(5400,9565)
(5149,9175)
(4970,8325)
(4850,7561)
(4642,6802)
(3677,6097)
(2716,5911)
(1600,5872)
```

R-Z 截面外的区域全部按不可达处理。

## 判定方法

一个点必须同时满足两件事，才算可达：

- 在 XY 平面的可达区域内
- 在 R-Z 垂直截面的可达区域内

一个长方体必须整体都在这个交集空间内，才会被判定为“在范围内”。

工具会检查长方体角点和表面采样点，避免只看角点导致误判。

## 文件说明

当前网页主文件是：

```text
outputs/GL-3DPRT-SP-M.html
```

设备选择页是：

```text
outputs/index.html
```

说明文档是：

```text
outputs/GL-3DPRT-SP-M说明.md
```

项目级说明是：

```text
README.md
```

维护辅助文件：

```text
outputs/glb-compressor.html

```

其中 `glb-compressor.html` 用于压缩和导出 GLB 模型，`preview.html` 用于界面预览和对照，不是正式设备入口。

分享给别人时，如果需要显示设备模型，请同时提供 SP-M 的 HTML 文件和 `outputs/assets` 文件夹。至少要包含：

- `outputs/assets/GL-3DPRT-SP-M.glb`：用于 GitHub Pages 或本地 HTTP 服务访问。
- `outputs/assets/GL-3DPRT-SP-M-model.js`：用于直接双击 HTML、以 `file://` 方式打开。

如果要同时提供 SP-M 和 SP-S 两个入口，分享 `outputs/index.html`、两个设备 HTML 文件以及完整的 `outputs/assets` 文件夹。
