# Device Motion Range Workspace Skill

用途：生成和维护蜘蛛式建筑 3D 打印机运动范围判定工具。

## 核心判定方法

- 3D 可达空间 = X-Y 平面可达区域 与 R-Z 垂直截面可达区域 的交集。
- R-Z 截面外全部不可达。
- 如果 X-Y 平面存在内圆不可达区，按整个高度方向贯穿的不可达圆柱处理。
- 设备模型只作为视觉参考，不参与长方体可达性判定。

## 新设备需要的数据

- 设备型号名称。
- R-Z 垂直截面边界点。
- X-Y 平面可达区域参数。
- 内圆或其他区域是否不可达。
- L/W/H 最大值和默认尺寸。
- X、Y、Z 的固定规则和生长方向。
- 可选设备模型 GLB，以及模型位置、姿态、透明度。

## GLB 设备模型加载规则

页面打开时不加载设备模型。只有点击“显示设备模型”后才加载。

- `http:` / `https:` 访问：加载 `outputs/assets/<DEVICE>.glb`。
- `file:` 直接双击 HTML 打开：加载 `outputs/assets/<DEVICE>-model.js` 备用模型包。

备用模型包是经典 script 文件，内容写入：

```js
window.__GL3DPRT_MODELS__[DEVICE] = BASE64_GLB;
```

浏览器禁止 `fetch(file://...)`，所以本地双击 HTML 时必须使用这个备用包。

每次 GLB 更新后，都要同步重新生成对应的 `*-model.js`。

- 不显示坐标轴数字和单位。
