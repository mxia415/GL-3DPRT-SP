# 三页面中英切换设计

## 目标

为 `index.html`、`GL-3DPRT-SP-M.html` 和 `GL-3DPRT-SP-S.html` 增加完整的中英文界面切换。默认语言为中文；用户切换后，三个页面都沿用最后一次选择。

## 范围

- 首页标题、说明、设备入口卡片、版本信息和页脚。
- SP-M 与 SP-S 页面标题、版本信息、STL 参考模型区、内接长方体区、尺寸和状态 HUD、视角按钮、设备显示按钮。
- 所有由脚本运行时更新的文字，包括 STL 状态、交线按钮、长方体显示状态、范围判定结果、尺寸单位和角点统计。
- 每个页面都在现有标题区域增加一个中英切换按钮，不改变当前单主题视觉风格和 3D 判定逻辑。

## 交互与状态

- 初始语言使用 `zh`。
- 切换按钮显示当前可切换目标：中文界面显示 `EN`，英文界面显示 `中文`。
- 语言值写入 `localStorage` 的 `gl3dprt-lang` 键；三个页面加载时读取同一键。
- 切换语言后立即刷新静态文本、按钮文本、页面标题和当前动态状态，不重置 STL、长方体尺寸、视角或 3D 场景。
- 如果本地存储不可用，页面仍使用中文并允许当前页面临时切换。

## 技术方案

- 每个独立 HTML 内保留自己的最小翻译字典和 `t(key, values)` 辅助函数，避免新增构建步骤和外部依赖。
- 静态文本使用 `data-i18n`、`data-i18n-title` 和 `data-i18n-aria-label` 标记；动态文本统一通过翻译函数生成。
- 动态数字继续使用现有格式化逻辑，只替换周围的语言文案。
- 使用当前项目的 CSS 控件样式，使语言按钮与现有页面按钮保持一致。

## 版本与文档

由于这是用户可见功能变更，交付版本更新为 `V2.0 · 2026-07-17 · ©Ming Xia`，同步更新三个 HTML 的标题和可见版本信息，以及两份设备说明文档中的版本元数据和语言使用说明。

## 验证标准

- 三个页面均存在并可点击语言按钮。
- 首页切换到英文后，进入 SP-M 或 SP-S 页面仍为英文；切回中文后三个页面恢复中文。
- 设备页切换语言不会清除或重建当前 3D 场景，隐藏长方体、显示设备模型、显示交线等状态按钮仍准确。
- 默认 3D 场景、动态判定状态、尺寸显示和视角控制均无浏览器控制台错误。
- 桌面视口下语言按钮、侧栏和 3D 区域不发生重叠或横向溢出。
