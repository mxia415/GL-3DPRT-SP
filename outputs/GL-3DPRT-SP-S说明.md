# GL-3DPRT-SP-S 运动范围判定说明

版本：v1.10

日期：2026-06-23

## 这个工具是做什么的

这个工具用来判断一个长方体能不能放进设备的运动范围里。

简单说：

- 绿色透明区域表示设备能到达的空间。
- 黄色长方体表示你输入的尺寸。
- 如果长方体完全在绿色范围内，就显示“长方体在范围内。”
- 如果有一部分超出去，就会提示不可达。

## 怎么看这个 3D 图

- 透明绿色体：设备可达范围
- 黄色长方体：当前输入的长方体
- 红色长方体：当前尺寸或位置不可达
- 淡黄色设备模型：设备外形参考，只是帮助看方向和位置，不参与判定
- X 方向：左右方向
- Y 方向：前后方向
- Z 方向：高度方向
- Z=0：地面

鼠标操作：

- 拖动鼠标：旋转视角
- 滚轮：放大或缩小
- 右上角“视图”：快速切换默认、前、上、右视图

网页默认先不加载设备模型，这样 GitHub Pages 打开速度更快。需要查看设备外形时，点击左侧的“显示设备模型”；加载完成后按钮会变成“隐藏设备模型”，再点一次可以隐藏。

当前网页使用按需加载的设备模型文件。设备模型仅作为半透明参考显示，位置为 `X=0, Y=2000, Z=0`，透明度为 `0.15`。

如果通过 GitHub Pages 或本地 HTTP 服务访问网页，按钮会加载 `outputs/assets/GL-3DPRT-SP-S.glb`。如果直接双击 HTML 文件、本地以 `file://` 方式打开，浏览器不能直接读取 GLB 文件，网页会改为加载 `outputs/assets/GL-3DPRT-SP-S-model.js` 备用模型包。

## 左侧参数怎么填

### X 向位置

X 方向已经固定在 `0`，也就是长方体左右居中放置。

这个值不能修改。

### Y 向内侧边位置

这个值表示长方体靠近设备内侧的那一条边在 Y 方向的位置。

- 如果 Y 是正数，长方体从这个位置向 Y+ 方向展开。
- 如果 Y 是负数，长方体从这个位置向 Y- 方向展开。

### 底面高度 Z

底面高度固定为 `0`，也就是长方体放在地面上。

长方体只会向上展开。

### 长度 L

长方体在 X 方向的尺寸。

最大值：`12000 mm`

### 宽度 W

长方体在 Y 方向的尺寸。

最大值：`6000 mm`

### 高度 H

长方体在 Z 方向的尺寸。

最大值：`7400 mm`

## 可达范围是怎么来的

可达范围由两部分共同决定。

第一部分是侧面范围，也就是 R-Z 垂直截面。

可以理解为：从侧面看，设备能到达的高度和远近范围。

第二部分是俯视范围，也就是 X-Y 平面范围。

可以理解为：从上往下看，设备能覆盖到哪些地方。

只有同时满足侧面范围和俯视范围，才算真正可达。

这个判定方法也可以用于其他设备。不同设备之间通常只需要更换两类实测数据：

- R-Z 垂直截面范围，也就是侧面看到的“远近 R”和“高度 Z”的边界点。
- X-Y 平面范围，也就是俯视看到的可达区域，比如外圆、内圆、偏心圆或其他边界。

换设备时，判定逻辑不变：每个点都必须同时在 X-Y 平面范围内，也必须在 R-Z 垂直截面范围内。

## 为什么中间有一块不可达

俯视范围里有一个内圆区域，这个区域不能进入。

参数如下：

- 外圆圆心：`(0, 0)`
- 外圆半径：`7429 mm`
- 内圆圆心：`(0, -786)`
- 内圆半径：`3407.5 mm`

长方体不能碰到内圆不可达区域。

## 如果要做其他设备

需要准备下面这些信息：

1. 设备型号名称。
2. R-Z 垂直截面边界点，格式为 `(R, Z)`。
3. X-Y 平面可达范围，比如外圆、内圆、圆心和半径。
4. 哪些区域是不可达的，比如内圆是否为不可达区域。
5. 长方体默认尺寸和最大 L/W/H。
6. Y 方向从哪里开始计算，以及正负方向如何生长。
7. 如果需要显示设备外形，还需要提供 STL 或 GLB 模型，以及模型在坐标系中的位置和姿态。

只要这些数据明确，就可以用同一套方法生成新的 3D 运动范围判定工具。

## 三个最大贴合按钮怎么用

### 固定L最大贴合

适合你已经确定长度 L，不想改长度的时候使用。

点击后：

1. L 保持不变。
2. 先找最大的 W。
3. 再在这个 W 下找最大的 H。

### 固定W最大贴合

适合你已经确定宽度 W，不想改宽度的时候使用。

点击后：

1. W 保持不变。
2. 先找最大的 L。
3. 再在这个 L 下找最大的 H。

### 固定H最大贴合

适合你已经确定高度 H，不想改高度的时候使用。

点击后：

1. H 保持不变。
2. 先计算从 `Z=0` 到目标高度之间共同可用的 R-Z 半径范围。
3. 在这个共同范围内搜索 L 和 W，并优先选择底面积 `L×W` 更大的组合。
4. 避免生成 `W=10 mm` 这类很窄、但实际不好用的极端解。

## 最大贴合时 Y 的规则

固定 L 或固定 W 最大贴合时：

- 当前 Y 是正数：从内圆正向边界开始，当前参数下为 `Y=2622`，向 Y+ 方向展开。
- 当前 Y 是负数：从内圆负向边界开始，当前参数下为 `Y=-4194`，向 Y- 方向展开。

固定 H 最大贴合时：

- 当前 Y 是正数：从内圆正向边界开始，并结合当前高度下共同可用的 R-Z 半径范围，寻找底面积更大的 L/W 组合。
- 当前 Y 是负数：从内圆负向边界开始，并结合当前高度下共同可用的 R-Z 半径范围，寻找底面积更大的 L/W 组合。

这里的内圆边界不是手动写死的，会根据内圆圆心和半径自动计算：

- 正向边界：`-786 + 3407.5 = 2621.5`，取整后为 `2622`
- 负向边界：`-786 - 3407.5 = -4193.5`，取整后为 `-4194`

## 重置按钮

### 重置尺寸

恢复默认立方体：

- X=0
- Y=2622
- Z=0
- L=1200
- W=1200
- H=1200

### 重置视图

只恢复观察视角，不改变长方体尺寸。

### 隐藏设备模型 / 显示设备模型

只控制中间淡黄色设备模型是否显示。

这个设备模型只是参考外形，方便确认方向和相对位置；它不会参与可达范围判定，也不会影响最大贴合结果。

当前网页使用按需加载的设备模型文件。页面打开时不会下载或解析模型；点击“显示设备模型”后才会加载。

通过 GitHub Pages 或本地 HTTP 服务访问时，网页加载 `.glb` 模型文件。直接双击 HTML、以 `file://` 打开时，网页加载 `*-model.js` 备用模型包，用来避开浏览器对本地 `fetch(file://...)` 的限制。

## 结果怎么看

常见结果有三种：

- `长方体在范围内。`  
  表示当前长方体可以放进可达范围。

- `长方体超出 R-Z 截面或 X-Y 可达区域。`  
  表示长方体有一部分超出了设备运动范围。

- `长方体与内圆不可达区域相交。`  
  表示长方体碰到了中间不可达区域。

## 文件说明

设备选择页是 `outputs/index.html`。

当前 SP-S 网页主文件是 `outputs/GL-3DPRT-SP-S.html`。

如果要分享单个设备页面，优先分享 `outputs/GL-3DPRT-SP-S.html`；如果需要显示设备模型，请同时提供 `outputs/assets` 文件夹。至少要包含：

- `outputs/assets/GL-3DPRT-SP-S.glb`：用于 GitHub Pages 或本地 HTTP 服务访问。
- `outputs/assets/GL-3DPRT-SP-S-model.js`：用于直接双击 HTML、以 `file://` 方式打开。

如果要同时提供 SP-M 和 SP-S 两个入口，分享 `outputs/index.html`、两个设备 HTML 文件以及完整的 `outputs/assets` 文件夹。说明文档是 `outputs/GL-3DPRT-SP-S说明.md`。
